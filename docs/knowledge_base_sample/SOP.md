# SOP

## 1. 密码重置处理流程
1. 核验用户身份，确认申请人和账号持有人一致。
2. 检查账号是否处于锁定、冻结或停用状态。
3. 若支持自助重置，指导用户通过“忘记密码”流程完成重置。
4. 若无法自助重置，由管理员在后台执行密码重置。
5. 重置后通知用户首次登录及时修改密码。
6. 将处理结果写入工单备注。

## 2. 权限申请处理流程
1. 确认申请系统名称、目标角色和申请原因。
2. 检查是否提交了直属主管审批信息。
3. 若信息不完整，要求用户补充材料。
4. 审批通过后，由 IT 支持或系统管理员开通权限。
5. 开通后更新工单状态，并通知用户验证是否生效。
6. 若权限涉及敏感模块，需额外人工复核。

## 3. VPN 故障排查流程
1. 确认用户使用的客户端版本和操作系统。
2. 检查网络是否可访问外网。
3. 检查账号状态、证书状态和密码是否正确。
4. 若提示证书错误，重新下发证书。
5. 若提示认证失败，检查账号是否过期或权限不足。
6. 若仍无法解决，升级给网络组处理。

## 4. 退款申请处理流程
1. 核实订单号、购买时间、产品类型和退款原因。
2. 检查是否符合退款条件。
3. 若可自动退款，则进入财务或支付系统处理。
4. 若涉及人工审核，则生成退款工单并提交审核。
5. 审核通过后记录处理结果。
6. 向用户说明预计到账时间和注意事项。

## 5. 系统故障升级流程
1. 记录故障时间、用户范围、错误信息和操作步骤。
2. 判断是单用户问题还是系统级问题。
3. 若影响范围较大，升级为高优先级工单。
4. 通知对应技术负责人排查。
5. 收集日志、截图和复现步骤。
6. 故障恢复后发送处理结果和后续说明。

## 6. 工单分派规则
1. 账号、权限、密码问题分派给 IT 支持组。
2. 网络、VPN、终端问题分派给网络与设备组。
3. 退款、支付问题分派给财务或售后组。
4. 产品功能咨询分派给产品支持组。
5. 系统报错和接口异常分派给研发或运维组。
6. 若分类不明确，先进入待分派状态。

## 7. SLA 处理规则
1. P0 工单需要优先响应，尽快进入处理状态。
2. P1 工单应在一个工作小时内响应。
3. P2 工单应在一个工作日内响应。
4. P3 工单可在常规支持周期内处理。
5. 若需用户补充材料，工单状态改为等待用户补充。
6. 超过 SLA 时间需触发提醒。

## 8. 知识库更新流程
1. 收集新问题、新答案和新流程。
2. 检查是否与现有 FAQ 或 SOP 重复。
3. 由知识管理员审核内容。
4. 更新知识文档后重新切分并入库。
5. 保留版本号和更新时间。
6. 对高频问题优先补充。
