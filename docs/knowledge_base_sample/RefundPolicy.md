# Refund Policy

## 1. 适用范围

本政策适用于 SupportOps Agent 演示知识库中的退款与售后处理场景，主要用于模拟企业客服、售后和财务审核流程。

## 2. 可退款场景

1. 用户在试用期内申请退订；
2. 重复扣费或重复下单；
3. 服务未开通但已完成支付；
4. 因系统故障导致无法正常使用；
5. 购买的套餐与实际需求不符且在可退期限内。

## 3. 不可退款场景

1. 服务已正常开通并使用较长时间；
2. 超过退款期限；
3. 用户未提供必要订单信息；
4. 属于定制化、不可二次销售的特殊服务；
5. 因用户自身操作失误导致的问题，但不符合退款条件。

## 4. 退款申请流程

1. 用户提交订单号、购买时间、退款原因；
2. 客服核验订单状态和退款条件；
3. 若符合条件，创建退款工单；
4. 若需要财务审核，提交人工审批；
5. 审批通过后执行退款；
6. 将处理结果通知用户并归档工单。

## 5. 审核要点

- 是否在退款时效内；
- 是否提供完整订单信息；
- 是否存在已使用服务的情况；
- 是否存在重复退款风险；
- 是否需要财务复核。

## 6. 退款说明模板

### 6.1 可以退款时
您的退款申请已受理，当前正在审核中。若审核通过，预计将在 3 到 5 个工作日内原路退回。

### 6.2 暂不支持退款时
根据当前订单状态和退款政策，该订单暂不符合退款条件。若您认为存在异常，请补充订单号和相关证明材料，我们会进一步核查。
